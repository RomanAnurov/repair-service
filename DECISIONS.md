# DECISIONS.md — Ключевые архитектурные решения

## 1. Бэкенд: Express + Sequelize
**Почему:** Минимальный boilerplate, отличная поддержка MySQL, встроенные миграции и валидация. Альтернативы (NestJS, Fastify) избыточны для объёма ТЗ.

## 2. Фронтенд: React + Vite + SCSS (BEM)
**Почему:** Быстрая сборка, простые и точные стили без CSS-in-JS, соответствует требованию "precise and simple style settings".

## 3. База данных: MySQL в Docker
**Почему:** Соответствует ТЗ ("можно SQLite или MySQL/Postgres"), легко воспроизводится локально через docker-compose, поддержка UTF8MB4 для кириллицы.

## 4. Миграции: Sequelize CLI
**Почему:** Версионирование схемы БД, возможность отката, стандарт индустрии. Альтернатива `sequelize.sync()` не подходит для продакшена.

## 5. Защита от гонок: WHERE-условие + проверка affectedRows
**Почему:** Атомарное обновление в БД надёжнее и проще, чем отдельные lock-механизмы. Возврат 409 Conflict при конфликте — стандарт REST API.

## 6. Авторизация: упрощённая (выбор пользователя)
**Почему:** Тестовое задание не требует полноценной auth-системы с JWT/сессиями. Сиды с пользователями достаточно для демонстрации ролевой логики.

## 7. Тесты: Jest + Supertest
**Почему:** Минимальный набор для проверки критической логики (гонки, CRUD), стандарт для Node.js, быстрое выполнение в CI/CD.
