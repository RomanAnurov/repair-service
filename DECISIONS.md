# DECISIONS.md — Ключевые архитектурные решения

## 1. Бэкенд: Express + Sequelize
**Почему:** Минимальный boilerplate, отличная поддержка MySQL, встроенные миграции и валидация.

## 2. Фронтенд: React + Vite + SCSS (BEM)
**Почему:** Быстрая сборка, простые и точные стили без CSS-in-JS, соответствует требованиям ТЗ.

## 3. Авторизация: упрощённая (выбор пользователя)
**Почему:** Тестовое задание не требует полноценной auth-системы. Сиды с пользователями достаточно.

## 4. Защита от гонок: WHERE-условие + проверка affectedRows
**Почему:** Атомарное обновление в БД надёжнее и проще, чем отдельные lock-механизмы. Возврат 409 при конфликте.

## 5. State management: Context API
**Почему:** Достаточно для объёма задачи, избегаем избыточного Redux/Zustand.

## 6. БД: MySQL в Docker
**Почему:** Соответствует ТЗ, легко воспроизводится локально через docker-compose.

## 7. Тесты: Jest + Supertest
**Почему:** Минимальный набор для проверки критической логики (гонки, CRUD), стандарт для Node.js.
